# Side Quest Studios - Deep Content Strategy

## Brand Identity

**Who We Are:**
Two friends who got tired of talking about building things and actually started building them.

**Core Values:**
- Obsession with shipping
- Love for clean code
- Hatred for meetings that could have been emails
- Build tools for ourselves → share them → fix them if they suck

**Voice:**
- No corporate speak
- Direct, sometimes irreverent
- Honest about the mess
- Anti-bureaucracy
- Maker-first

- Share their journey on X

- Escaped FAANG/digital agencies
- Prefer autonomy
- Small teams, big impact

## Content Pillars (5 Pillars)

### Pillar 1: Ship Fast (30%)
**Hook:** "We shipped X in Y time."

**Topics:**
- Rapid prototyping
- MVPs, not polished products
- Deadlines create focus
- Iteration over perfection
- Shipping on Fridays

**Sample Hooks:**
- "Shipped a complete feature in 4 hours. Here's what I learned."
- "The prototype works. Now the real work begins."
- "Three features became one. The code is better for it."
- "Friday ship day. Every Friday."

---

### Pillar 2: Clean Code (25%)
**Hook:** "Here's how we write code that doesn't suck."

**Topics:**
- Simplicity over clever
- Delete more code
- Code reviews that don't suck
- Tests that actually help
- Refactoring without fear

**Sample Hooks:**
- "The best code is the code you don't write."
- "We deleted 500 lines. The app is better."
- "Your linter is not your enemy."
- "Complexity is the enemy. Kill it."

---

### Pillar 3: Anti-Corporate (20%)
**Hook:** "We don't do things the enterprise way."

**Topics:**
- No standups, no retrospectives
- No Jira for small teams
- Meetings that could be emails
- Bureaucracy-free development
- Small team autonomy

**Sample Hooks:**
- "No standups. No retrospectives. Just code."
- "Your startup doesn't need Jira. It needs a keyboard."
- "We don't do enterprise. We build for users."
- "The enterprise way is the slow way."

---

### Pillar 4: Founder Stories (15%)
**Hook:** "Here's the messy reality of building."

**Topics:**
- Revenue updates (when appropriate)
- User feedback we didn't expect
- Mistakes we made
- Wins worth celebrating
- The daily grind

**Sample Hooks:**
- "Users found a bug. We're excited. That's weird."
- "Started the day with coffee. Ended with a fix."
- "Revenue update: We made $X this month."
- "The feature nobody asked for but everyone loves."

---

### Pillar 5: Tool Picks (10%)
**Hook:** "We use these tools. They're good."

**Topics:**
- Tech stack reveals
- Library recommendations
- Hosting, database, payments
- Development workflow
- Productivity systems

**Sample Hooks:**
- "We've tried 12 auth solutions. This is the best."
- "Vercel + Supabase + Stripe = shipped."
- "The tool that saved us 20 hours/week."
- "Switched to X. Zero regrets."

## Content Templates

### Ship Fast Template
```
[Hook about speed/time]
[Brief context]
[One key lesson or insight]
[CTA/Question to audience]
```

Example:
```
Shipped a complete auth system in 3 hours.

We needed email login. Spun up Supabase Auth. Done.

Key insight: Don't build commodity infrastructure. Pay for it instead.

What infrastructure do you still build yourself?
```

---

### Clean Code Template
```
[Problem with complex code]
[Simple solution approach]
[Why simplicity wins]
[Actionable takeaway]
```

Example:
```
Your service layer doesn't need 15 abstraction layers.

We deleted 500 lines last week. The app loads faster. Bugs decreased.

Simplicity isn't dumb. It's discipline.

What code are you over-engineering?
```

---

### Anti-Corporate Template
```
[Corporate practice we reject]
[Our simpler alternative]
[Why it works better]
[Invitation to share]
```

Example:
```
No daily standups. No "synergy" meetings. No "alignment" sessions.

We have a Slack channel. We talk when there's something to say.

Our team is 2 people. We don't need bureaucracy.

What corporate ritual are you skipping?
```

---

### Founder Story Template
```
[Specific moment from the week]
[What we learned]
[How it changes our approach]
[Open question]
```

Example:
```
A user refunded this week. First one in 3 months.

It stings. But they were right—the onboarding was confusing.

We're rewriting that flow today.

How do you handle negative feedback?
```

---

### Tool Pick Template
```
[Problem we had]
[Tool we tried]
[Why it worked for us]
[Who should try it]
```

Example:
```
We tried 12 auth solutions. Clerk is the best.

The dev experience is incredible. Pre-built components. 
Documentation that doesn't suck.

Building a SaaS? Start there.
```
